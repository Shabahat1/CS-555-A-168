Security-Usability Intersection

One of the most critical challenges in cybersecurity is the security-usability paradox: the stronger a security measure, the more friction it introduces into the user experience. Norman's seven stages of action provide a cognitive framework for understanding where users encounter gulfs of execution and evaluation in security workflows.

For example, consider enabling Wi-Fi on a smartphone. A naive implementation may hide the Wi-Fi network list behind multiple taps (gulf of execution), making users resort to turning airplane mode on and off instead. A Norman-informed design shortens the distance between intention and execution—e.g., a quick-settings tile with instant network discovery.

Similarly, GOMS analysis helps us decompose expert user tasks (e.g., a penetration tester configuring a proxy) into operators (keystroke, pointer, mental operations), revealing bottlenecks where KLM timing predictions suggest that optimizing a single menu structure saves seconds per action and hours per day at scale.

By applying rigorous HCI methodology to security engineering, we can build systems that protect users without sacrificing usability—a principle central to my research and professional work.
