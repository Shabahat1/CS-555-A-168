10 Common HTML Failures in HCI

From an HCI perspective, code errors lead to accessibility and usability barriers.

1. Unlabeled Form Controls
Error: <input type="text"> (No context for screen readers).
Fix: <label for="id">Label</label>
2. Missing 'Alt' Attributes
Error: <img src="icon.png">
Fix: Add alt="Description" for perception (Dix, Ch. 1).
3. Improper Heading Hierarchy
Error: Using <h4> because of its default size without an <h1>.
Fix: Use headings to define semantic structure, not style.
4. Broken Links
Error: <a href="#">Click Here</a> (Vague intent).
Fix: Describe the destination clearly.
5. Duplicate IDs
Error: Multiple elements with id="nav".
Fix: IDs must be unique to prevent script/CSS failures.
6. Non-Semantic HTML
Error: Using <div> for everything.
Fix: Use <nav>, <main>, <article>.
7. Missing Language Attribute
Error: <html>
Fix: <html lang="en"> (Critical for screen readers).
8. Poor Contrast
Error: Light grey on white background.
Fix: Ensure WCAG 2.1 contrast ratios.
9. Fixed-Width Tables for Layout
Error: <table width="800">
Fix: Use tables only for data, not layout.
10. Interactive Elements not Keyboard Focusable
Error: <span onclick="...">Button</span>
Fix: Use <button> or tabindex="0".
