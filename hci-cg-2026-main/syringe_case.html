The Scenario (Dix, Ch. 3)

In clinical settings, an automatic syringe pump is used to deliver precise doses of medication. A common design failure occurs when the user (a nurse or doctor) attempts to set a specific dosage, but the interface creates a "Gulf" that leads to potentially fatal errors.

1. The Gulf of Execution

Definition: The gap between the user's goal and the physical actions required by the system.

Execution Failures in the Syringe Pump:

  • Poor Mapping: If the user wants to increase the dose from 5ml to 10ml, but the interface uses a single button that cycles through values (1, 2, 5, 10, 20), the user must perform multiple cognitive steps to "Specify" the action.
  • Input Complexity: If the "Increase" button is too sensitive or has no tactile feedback, it is difficult to "Execute" the exact stopping point.

2. The Gulf of Evaluation

Definition: The gap between the system's state and the user's ability to perceive and interpret it.

Evaluation Failures in the Syringe Pump:

  • Inadequate Feedback: The display may show a small "10" without specifying the unit (is it 10ml or 10mg?). This creates a failure in the "Interpret" stage of Norman's cycle.
  • Perceptual Latency: If the screen updates slowly while the dose is being set, the user cannot "Perceive" the change in real-time, causing them to overshoot the intended goal.

HCI Lessons Learned

To bridge these gulfs, designers must ensure Visibility of System Status and Natural Mapping. In safety-critical systems like a syringe pump, errors are often "System Errors" disguised as "User Errors."
