Norman vs. GOMS: Secure Banking Login

This analysis applies Norman's seven stages of action and GOMS to a security-critical task: authenticating on a banking website using two-factor authentication (2FA). Understanding both models helps security engineers design login interfaces that are both secure and usable.

Comparison Table

Comparison of Norman's stages and GOMS concepts applied to secure banking login with 2FA
Norman's Stage Real-World Example (Banking Login with 2FA) GOMS Interpretation Alignment & Security Insight
Goal (User Intention) "I need to access my bank account securely." Goal: Authenticate-to-Banking-System ✓ Perfect Match – Both models identify the primary goal. Users may not understand 2FA's necessity.
Plan (Decision & Strategy) "I'll enter username, password, then use my authenticator app." Method & Selection Rules: Method 1: Password + App; Method 2: Password + SMS ✓ Conceptual Match – Clear UI labels reduce cognitive load.
Specify (Perceptual Mapping) Visual search: Locate username field, password field, "Send Code" button, security icons. Not Explicitly Modeled in GOMS. Key Divergence – Missing security indicators increase gulf of execution. Phishing exploits this.
Execute (Physical Action) Type username, password, click "Send Code," type 6-digit code. Operators: [K] keystroke, [P] point/click, [M] mental prep ✓ Direct Match – KLM timing shows 2FA adds significant friction.
Perceive & Interpret (Feedback) "Code sent to your phone" message; authenticator app shows code. Not Modeled in GOMS. Fundamental Divergence – Delays create uncertainty (gulf of evaluation).
Evaluate (Goal Achievement) Dashboard loads; user confirms secure login. Not Modeled in GOMS. Fundamental Divergence – Phishing sites can "succeed" at this stage. Clear confirmation UI needed.

Key Security-HCI Insights

  • Gulf of Execution: Users don't understand why 2FA is required, leading to workarounds.
  • Gulf of Evaluation: Ambiguous feedback creates uncertainty. Clear, timely messages close this gap.
  • KLM Timing: Optimizing login speed (e.g., biometric 2FA) improves security adoption.
  • Visual Cues: Security indicators must be salient to prevent spoofing attacks.
